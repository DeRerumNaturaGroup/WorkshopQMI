The Problem of Representation

  • Overview
  • 🟣 This session…

  • Question 1: How do we represent a molecule/crystal?

    In school we were taught to write formulas (CO\(_2\), C\(_3\)H\(_8\) etc) and draw Lewis structures. These are idealizations.

    A molecule is neither a string of letters, nor a picture (just as a person is neither a name nor a picture). A molecule is a collection of atoms in 3D space.

  • Question 2: How do we represent a molecule/crystal for a computer?

    We can do that using a Crystallographic Information Framework (CIF) file for instance.

  • Question 3: Is this a good representation for machine learning?

    No. Because it is not invariant to translation, rotation, and permutation of atoms. We need a representation that is invariant to these transformations.

Two representations that are commonly used in cheminformatics are SMILES strings and Molecular fingerprints

This session is about data representations and we will focus on the following topics:

1 The SMILES notation for molecules

2 Molecular fingerprints

We will also discuss how we can use these representations to perform ask questions such as:

3 Does this molecule contain this substructure? (i.e. is this substructure present in this molecule)

4 Is this molecule similar to that molecule? (i.e. how similar are two molecules)

SMILES

What is SMILES Rules for SMILES Code Implementation

SMILES (Simplified Molecular Input Line Entry System) is a chemical notation that allows a user to represent a chemical structure as a string of character in a way that can be used by the computer.

The SMILES for ethanol is “CCO”:

  • The C’s represent carbon, and the O represents oxygen.
  • Bonds are implied by the order of the characters.
  • Note there are no H atoms in the string.

SMILES has five basic syntax rules – If basic rules of chemistry are not followed in SMILES entry, (for example, if a moleculs has too many bonds on an atom), a warning will appear that the structure is impossible.

Let’s see the rules and some examples

Warning SMILES is not a unique representation. Ethanol can also be written as “C(C)O” or “OCC”. All of these strings represent the same molecule.

    • Upper case letters refer to non-aromatic atoms; lower case letters refer to aromatic atoms.

  • Single bonds are the default and therefore need not be entered. ‘CC’ means a non-aromatic carbon is attached to another non-aromatic carbon by a single bond, and the computer would identify the structure as the chemical ethane.
  • Bond between two lower case atom symbols is aromatic.
  • A blank terminates the SMILES string.
Jargon Aromaticity

is a chemical property in which a molecule has a cyclic structure with alternating single and double bonds. Aromatic compounds are stable and have unique chemical properties. Aromaticity is often associated with the presence of benzene rings, which are six-membered carbon rings with alternating single and double bonds.

Combining atomic symbols and bond symbols yields simple chain structures.

By default hydrogen is not written but can be added explicitly

The SMILES interpreter understands the number of possible connections that an atom can have. If enough bonds are not present, the system will automatically assumes the rest are hydrogens.

Hydrogen bonds can be added explicitly, but if one hydrogen bond is identified in the string, the SMILES interpreter will assume that all hydrogens have been added. + HC(H)=C(H)(H) Ethene

Warning Beware of chemicals with two letters.

  • ‘Sc’ could be interpreted as:

    • a sulfur atom connected to an aromatic carbon by a single bond,
    • or it could be the symbol for scandium.

  • The SMILES interpreter gives priority to the interpretation of a single bond connecting a sulfur atom and an aromatic carbon.

  • To identify scandium the user should enter [Sc].

A branch from a chain is specified by placing the SMILES symbol(s) for the branch between parenthesis.

The string in parentheses is placed directly after the symbol for the atom to which it is connected.

If it is connected by a double or triple bond, the bond symbol immediately follows the left parenthesis.

SMILES allows a user to identify ring structures by using numbers to identify the opening and closing ring atom. For example, in C1CCCCC1, the first carbon has a number ‘1’ which connects by a single bond with the last carbon which also has a number ‘1’. The resulting structure is cyclohexane. Chemicals that have multiple rings may be identified by using different numbers for each ring. If a double, single, or aromatic bond is used for the ring closure, the bond symbol is placed before the ring closure number. Some examples:

Charges on an atom can be used to override the knowledge regarding valence built into SMILES system.

The charged atom is followed by brackets which enclose the charge on the atom. The number of charges may be explicitly stated ({-1}) or not ({-}).

Molecular Fingerprints

Molecular Fingerprints

What are they?

MACCS ECFP

Deep learning models almost always take arrays of numbers as their inputs. But how can we turn a molecule into numbers.

Many (but not all) types of models require their inputs to have a fixed size. This can be a challenge for molecules, since different molecules have different numbers of atoms. If we want to use these types of models, we somehow need to represent variable sized molecules with fixed sized arrays.

Fingerprints are designed to address these problems.

A fingerprint is a fixed length array, where different elements indicate the presence of different features in the molecule. If two molecules have similar fingerprints, that indicates they contain many of the same features, and therefore will likely have similar chemistry.

Some common fingerprints are:

  • Molecular ACCess System keys or MACCS-keys
  • Avalon fingerprint
  • Atom-pair fingerprint
  • Topological-Torsions fingerprint.
  • Morgan fingerprint or Circular Fingerprint

MACCS

A chemical fingerprint is a list of binary values (0 or 1) which characterize a molecule. There are several ways to create the list.

MACCS (Molecular ACCess System) keys are a set of questions about a chemical structure, for instance:

  • Are there fewer than 3 oxygens?
  • Is there a S-S bond?
  • Is there a ring of size 4?
  • Is at least one F, Cl, Br, or I present?

The result of this is a list of binary values – either true (1) or false (0). This list of values for a given chemical structure is called the MACCS key fingerprint for that structure.

Example. For C1CCC1 then the answers to those questions are:

  • 0 oxygens < 3 oxygens → True
  • no S-S bond → False
  • there is a ring of size 4 → True
  • there are no halogens → False

The answers are frequently written as a list of bits (also called a bitstring). The bitstring for this molecule is 1010.

There are 166 public keys (fragment definitions) of MACCS in RDKit implementation. Essentially, it is a binary fingerprint (zeros and ones) that answer 166 fragment related questions.

Extended Connectivity FingerPrinting (ECFP) basically proceed as follows (Source) . For EACH atom:

Benzamide molecule with atom 6 being featurized.

Step 1 - Initial Stage: The atom in the molecule is assigned a unique integer identifier, is generated by hashing a combination of properties (atomic numbers, atomic mass, etc.)

Step 2 - Update: Each atom collects its identifier and the identifiers of its immediately neighboring atoms, into an array. A hash function is applied to reduce this array back into a new, single-integer identifier. This step is done to capture the neighborhood of the atom(an atom’s behavior is also influenced by the type of atoms that surrounds it).

Once all atoms have generated their new identifiers, the old identifiers are replaced with the new ones.

Step 3 - De-duplication Stage: The third stage removes the duplicate features from our generated feature list (Beyond our present scope. See link above).

Step 4 - Forming the bit array: Once all the identifiers are calculated for a specified number of iterations, the final step is to reduce these identifiers into a bit array (Again beyond scope).

Molecular Similarity

Molecular Similarity

Overview Tanimoto Code

Molecular similarity is a concept in chemistry and cheminformatics stating that molecules with similar structures or chemical properties tend to exhibit similar biological activities and behaviors. It is a foundational principle used to discover new drugs, predict chemical properties, and explore “chemical space”.

Instead of comparing molecules visually, computers quantify structural and physicochemical resemblance through specific steps:

  • Molecular Representation: Molecules are converted into simplified mathematical formats like:
    • Molecular Fingerprints: Binary vectors (lists of 1s and 0s) that denote the presence or absence of specific atomic structures, rings, or functional groups.
    • Graphs/3D Pharmacophores: Representations focusing on molecular shape, spatial arrangement, and electrostatic potential.
  • Similarity Coefficients: Algorithms compare these representations to output a similarity score (typically between 0 and 1). The most common mathematical metric used is the Tanimoto coefficient.
